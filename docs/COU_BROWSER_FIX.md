# COU 浏览器用户体验修复

## 问题描述

从条款管理点击"查看COU"跳转到COU管理后：
1. ❌ 页面没有显示当前是哪个条款的COU
2. ❌ 无法切换到查看所有COU（包括其他条款的COU）

## 修复内容

### 1. 添加当前条款信息提示条

当从条款管理跳转过来时，在页面顶部显示紫色提示条，包含：
- 来源政策名称
- 章节和条款号
- 条款内容摘要
- 关闭按钮（可清除筛选）

```
┌─────────────────────────────────────────────────────────────────┐
│  📄 当前查看条款：个人信息保护法  第二章 第一节  第十三条       │
│     符合下列情形之一的，个人信息处理者方可处理个人信息...     │
│                                                            [×] │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 添加"显示全部COU"按钮

在顶部工具栏右侧添加"显示全部COU"按钮：
- 仅在处于条款筛选模式时显示
- 点击后清除条款筛选，显示所有COU
- 按钮样式：紫色边框，文件搜索图标

### 3. 页面标题更新

副标题更新为：
- 正常模式：`共 1000 条COU`
- 筛选模式：`共 15 条COU (筛选自 1000 条) · 当前筛选条款关联COU`

## 代码变更

### COUBrowser.tsx

1. **导入新增**：`X`, `FileSearch` 图标
2. **新增状态**：`currentClause` - 根据 `clauseFilter` 获取当前条款信息
3. **新增函数**：`handleShowAll` - 清除条款筛选
4. **UI更新**：
   - 添加条款信息提示条（紫色背景）
   - 添加"显示全部COU"按钮
   - 更新副标题显示筛选状态

## 使用流程

```
条款管理 (ClauseEditor)
    ↓ 点击"查看COU"
COU管理 (COUBrowser)
    ↓ 显示紫色提示条
┌─────────────────────────────────────────────────────┐
│ 当前查看条款：个人信息保护法 · 第二章 · 第十三条     │
│ 符合下列情形之一的，个人信息处理者方可处理个人信息   │
└─────────────────────────────────────────────────────┘
    ↓ 两种方式继续
    ├── 点击"显示全部COU" → 查看所有条款的COU
    └── 点击"返回条款列表" → 回到条款管理
```

## 验证步骤

1. 进入后台管理 → 条款管理
2. 点击某条款的"查看COU"按钮
3. **验证**：跳转到COU管理后，顶部显示紫色提示条，包含：
   - 政策名称
   - 章节和条款号
   - 条款内容摘要
4. **验证**：顶部工具栏右侧显示"显示全部COU"按钮
5. 点击"显示全部COU"
6. **验证**：紫色提示条消失，显示所有COU（不再只筛选该条款）
7. **验证**：页面标题恢复为"共 1000 条COU"
